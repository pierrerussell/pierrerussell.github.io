Clean Architecture

Built with Domain-Driven Design principles and Clean Architecture:

  • Domain Layer: Core business entities and logic with no external dependencies
  • Application Layer: Use cases and orchestration, DTOs and service interfaces
  • Infrastructure Layer: External integrations (FMP API) and data persistence (EF Core)
  • API Layer: RESTful endpoints with proper DI configuration

Key Design Decisions

  • Repository pattern for data access abstraction
  • Gateway pattern for external API integration
  • No Unit of Work pattern - repositories handle SaveChanges directly
  • SQLite for simplicity with database serving as cache

Quantitative Algorithms

Modern Portfolio Theory (MPT)

The portfolio optimization uses Markowitz mean-variance optimization:

  • Minimize: σ²p - λ × μp
  • Where σ²p is portfolio variance and μp is expected return
  • Subject to: Σwi = 1 and 0 ≤ wi ≤ wmax

Value at Risk (VaR)

Three methods implemented:

  • Historical VaR: 5th percentile of historical returns
  • Parametric VaR: μ - zα × σ (assumes normal distribution)
  • Monte Carlo VaR: Simulate 10,000 scenarios using historical volatility

Backtesting Engine

Walk-forward analysis with proper handling of:

  • Transaction costs (bid-ask spread + commission)
  • Market impact modeling
  • No look-ahead bias

Behind the Scenes

This project combines software engineering excellence with quantitative finance knowledge.

Key Challenges & Solutions

1. Portfolio Optimization Performance

Challenge: Quadratic optimization with 100+ assets was slow

Solution: Implemented efficient matrix operations using Math.NET Numerics and cached covariance matrices

2. Real-time Market Data

Challenge: FMP API has 250 calls/day limit

Solution: Database-first approach with intelligent caching and batch fetching

3. Backtesting Accuracy

Challenge: Avoiding look-ahead bias in strategy testing

Solution: Strict temporal ordering and point-in-time data access patterns

4. Financial Visualizations

Challenge: Making complex concepts intuitive

Solution: Interactive charts with educational tooltips and progressive disclosure

Lessons Learned

  • Clean Architecture really shines in financial applications where business logic complexity is high
  • F# was perfect for the mathematical calculations - immutability and type safety prevented many bugs
  • Caching strategy is crucial when working with rate-limited APIs
  • Financial data requires careful handling of edge cases (splits, dividends, missing data)
